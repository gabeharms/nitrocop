## Rules

- Primary metric is king. Improved → keep. Worse/equal → discard.
- Don't thrash — if you keep reverting the same idea, try something structurally different.
- Simpler is better. Removing code for equal perf = keep.
- Be careful not to overfit to the benchmarks and do not cheat on the benchmarks.
- Read source files thoroughly before making changes.
